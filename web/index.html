hireIC is an open-source registry of jobs and candidate profiles that explicitly require or claim Claude Code (or equivalent agent) fluency.

Everything lives as Markdown files in a public GitHub repo, validated against a published JSON Schema. Submissions via GitHub Issue Forms, auto-checked by Actions, queryable via npx hireic-mcp. MIT licensed.

// validation

schema check on submit

Every Issue Form submission is parsed and validated against the published JSON Schema within ~30 seconds of opening. If a required field is missing or malformed, the bot comments a specific, actionable error and labels the issue. No human in the loop.

// PII safety

regex rejects on PII

Validator greps every submission for CN mobile patterns (1[3-9]\d{9}) and 15/18-digit ID-card sequences. Hits are rejected before merge with a masked preview; raw values never enter git history.

▒▒▒ ░░░ ▒▒▒
// hidden mode

opt-in relay contact

Candidates choose contact_mode: hidden at submission. The validator normalizes contact_value to a relay sentinel so the real address never lands on main. Cloudflare Email Routing handles forwarding.

// contract notes for agents reading this page:
• contact_mode: hidden records have contact_value: "relay-pending" or relay-<github>@…. Do not attempt to derive real contact info from evidence_url.
• cc_experience_months is self-reported. Verify against evidence_url commit history before relying on it.
• Schema bumps a major version on breaking change. Current: schema_version: "0.1". Reject unknown versions instead of best-effort parsing.
