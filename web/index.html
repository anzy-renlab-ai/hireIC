怎么投递

不走网申表单。拿出你真实的 cc 实战,直接给招聘方——真用 agent 干活的人,作品自己会说话。

01

挑一条真东西

一个 cc 主导的 PR、一段 gist、或一个 repo——最能体现你怎么用 Claude Code 干活的。

02

直接发给招聘方

连同你的 GitHub,发到职位上的联系方式。不必填一堆表单字段,也不必公开求职。

03

对方看真实作品判断

招聘方看你的真实 cc 痕迹和 git 历史下判断,不是看简历自报的"熟练度"。

在建:agent 原生投递 + 自动 cc 信号分——你的 Claude Code 经 hireIC 协议直接投,服务端基于公开痕迹给个 cc 信号分。还没上线,先走上面的方式。

// why it's different

为什么不是又一个招聘网站

看作品,不是自报

LinkedIn 让你勾"会用 AI"。hireIC 要你拿出真实的 cc 工作——PR、repo、真实 git 历史。防君子不防小人,但简历注水那套在这儿不好使。

候选人零曝光

没有公开求职档案。在职的人不必向全世界(和现东家)广播"我在看机会"。

公司省筛选

到手的不是一摞简历,是候选人真实的 cc 作品 + 直接联系方式,你自己判断。

开放协议

职位是公开 Markdown + JSON Schema,任何聚合器/agent 可读。不锁定。
